# 指令：分析指定文件夹下所有项目的技术栈与代码规范
请你作为一名资深的全栈技术架构师和代码规范评审专家，完成以下任务：

## 核心任务
1. 扫描我当前文件夹下的**所有项目**（每个文件夹是一个单独项目）
2. 针对每个独立项目，完成两项核心分析：
   - 技术栈分析：识别并列出项目使用的编程语言、框架、库、工具、数据库、构建工具、包管理器等核心技术栈信息
   - 代码规范分析：识别项目遵循的代码规范（如命名规范、缩进规则、注释规范、代码风格指南、设计模式使用、错误处理方式等）

## 输出要求
1. 按「项目名称」分组输出，分别输出${到项目文件夹名称}.md文件里，每个项目的分析结果包含以下结构化内容：
   - 项目基本信息：项目名称、主要功能（基于代码内容推断）
   - 技术栈清单，分大类列出，示例：
     ✅ 编程语言：Python 3.9
     ✅ 框架：Django 4.2
     ✅ 工具：pytest、black、flake8
     ✅ 数据库：MySQL
     ✅ 数据库框架：Django ORM
     ✅ 构建工具：Docker
     ✅ 包管理器：pip
     ✅ 定时任务：Celery
   - 代码规范分析
2. 最后输出一份「整体总结」到AGENTS.md文件：
   - 所有项目的技术栈共性与差异
   - 项目具体信息md文件的阅读指引

## 补充说明
- 分析时优先关注核心代码文件（如 .py/.js/.java/.vue/.ts 等），忽略日志、缓存、依赖包（如 node_modules、venv、target）等非核心文件
- 代码规范分析 以当前项目已存在的规范为主，列举一些通用的即可
- 输出语言为中文，用词专业但易懂，避免冗余
